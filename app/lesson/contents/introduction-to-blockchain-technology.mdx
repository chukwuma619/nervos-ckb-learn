# Introduction to Blockchain Technology

## Learning outcomes

By the end of this lesson, you will:
- Understand the core concept of a distributed ledger and how it differs from traditional databases
- Explain the role of consensus mechanisms in maintaining blockchain integrity
- Identify key cryptographic principles that secure blockchain data
- Recognize the differences between centralized and decentralized systems
- Understand the immutability property of blockchains and why it matters
- Identify when blockchain technology might be appropriate for a given use case

## What is a Blockchain?

A blockchain is a type of **distributed digital ledger** that records transactions across multiple computers in a way that ensures the entries cannot be altered retroactively. Unlike traditional databases managed by a central authority, blockchains distribute identical copies of the ledger across a network of computers (called **nodes**).

Think of a blockchain as a digital record book where:
- New entries (transactions) are grouped into "blocks"
- Each block is cryptographically linked to the previous one
- The entire history is maintained and visible to all participants

```
Block 1 → Block 2 → Block 3 → ... → Block N
```

Each block typically contains transaction data, a timestamp, and a cryptographic reference (hash) to the previous block, forming a "chain" of blocks—hence the name "blockchain."

## Distributed Ledgers vs. Traditional Databases

To understand the revolutionary nature of blockchain, let's compare it with traditional database systems:

| Traditional Database | Blockchain (Distributed Ledger) |
|----------------------|--------------------------------|
| Centralized control  | Distributed control            |
| Single authority     | No central authority           |
| Can modify/delete data | Immutable (append-only)      |
| Private/limited access | Transparent (in public blockchains) |
| Fast performance     | Usually slower but more secure |
| Trust required in central authority | Trustless system |

In a traditional database, a central administrator has complete control. They can add, modify, or delete records. Users must trust this administrator to maintain data integrity.

In contrast, a blockchain distributes this responsibility across the network. No single entity controls the data, and once information is recorded, it becomes extremely difficult to change.

### Notes
The term "trustless" doesn't mean there's no trust involved. Rather, it means trust is placed in the cryptographic system and protocol rules instead of a central authority. The need for trust is minimized through mathematical verification.

## How Blockchain Works: The Basics

When you want to make a transaction on a blockchain:

1. You initiate a transaction (e.g., sending cryptocurrency)
2. This transaction is broadcast to all nodes in the network
3. Nodes validate the transaction using predetermined rules
4. Valid transactions are grouped into a "block"
5. The network reaches **consensus** on the validity of the block
6. The new block is added to the chain
7. The transaction is now complete and permanently recorded

This process ensures that all participants in the network have an identical copy of the ledger, without requiring a central authority to verify transactions.

## The Role of Consensus Mechanisms

A **consensus mechanism** is a protocol that ensures all nodes in the network agree on the validity of transactions and the order in which they are added to the blockchain. It's how the network achieves agreement without a central authority.

The two most common consensus mechanisms are:

1. **Proof of Work (PoW)**: Nodes (called miners) compete to solve complex mathematical puzzles. The first to solve it gets to add the next block and receives a reward. This requires significant computational power and energy.

2. **Proof of Stake (PoS)**: Validators are selected to create new blocks based on the number of coins they "stake" (lock up as collateral). This approach is more energy-efficient than PoW.

Other mechanisms include Delegated Proof of Stake (DPoS), Proof of Authority (PoA), and Practical Byzantine Fault Tolerance (PBFT), each with its own advantages and trade-offs.

### Notes
Consensus mechanisms make it prohibitively expensive or impractical for malicious actors to take control of the network. For instance, to manipulate a PoW blockchain, an attacker would need to control more than 50% of the network's computing power (a "51% attack").

## Cryptographic Foundations of Blockchain

Blockchain security relies heavily on cryptographic principles:

1. **Hashing**: A cryptographic hash function converts data of any size into a fixed-size string of characters. The same input always produces the same hash, but changing even one character in the input produces a completely different hash. Blockchains use hash functions to:
   - Create unique identifiers for blocks
   - Link blocks together
   - Verify data integrity

2. **Public-Private Key Cryptography**: Users have two keys:
   - A **private key** (kept secret) used to sign transactions
   - A **public key** (shared openly) that others can use to verify the signature

3. **Digital Signatures**: These prove ownership and authorization of transactions without revealing the private key.

```
Transaction → Signed with Private Key → Verified with Public Key
```

These cryptographic elements ensure that only the rightful owner of assets can transfer them, and once recorded, transactions cannot be tampered with.

## Key Properties of Blockchain Technology

1. **Decentralization**: No single entity controls the entire network, reducing central points of failure and censorship.

2. **Transparency**: In public blockchains, all transactions are visible to everyone, creating an auditable history.

3. **Immutability**: Once data is recorded and confirmed by the network, it becomes extremely difficult to alter or delete.

4. **Security**: Cryptographic principles make the system secure against tampering and fraud.

5. **Redundancy**: Multiple copies of the ledger exist across the network, making the system resilient to failure.

## When to Use Blockchain Technology

Blockchain is not a solution for every problem. It's most valuable in scenarios where:

- Multiple parties need to share and update data
- These parties may not fully trust each other
- A verifiable, immutable record of transactions is required
- Disintermediation (removing middlemen) provides value
- Censorship resistance is important
